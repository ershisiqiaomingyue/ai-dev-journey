# AI Agent 转型学习路线（2026.06 → 2027.02）

> **时间跨度**：2026年6月 → 2027年2月（约 9 个月）  
> **每日投入**：工作日 1.5 小时，周末半天（3-4 小时）  
> **总学习时长**：约 360-400 小时  
> **目标**：具备独立开发 AI Agent 能力，可求职或接副业

---

## 🗓️ 第一阶段：基础与手写 Agent（6月 – 7月）
**目标**：跑通 API，手写一个能调用工具的简单 ReAct Agent。

### 第1周（6/1 – 6/7）—— API 与第一个调用
- [ ] 注册 DeepSeek / 阿里云百炼，获取 API Key（免费额度够用）
- [ ] 用 Python 写一个脚本：发一条消息，打印完整返回
- [ ] 学习 token、temperature、system prompt 的基本概念
- [ ] 核心产出：一个 `hello_agent.py`，能回答"你是谁"

### 第2周（6/8 – 6/14）—— 函数调用（Tool Use）
- [ ] 读懂官方 Function Calling 文档
- [ ] 定义一个"获取当前时间"的工具（无参数）
- [ ] 让模型自动判断并调用这个工具
- [ ] 核心产出：一个 CLI 工具，问"现在几点" → 返回真实时间

### 第3周（6/15 – 6/21）—— 手写最简单的 Agent 循环
- [ ] 理解 ReAct 模式（Reason + Act）
- [ ] 写一个 `while True` 循环：
  - [ ] 调用模型
  - [ ] 如果模型要求调用工具 → 执行工具 → 将结果加入对话 → 继续循环
  - [ ] 否则输出最终答案并结束
- [ ] 核心产出：一个"查天气 + 算加法"的多工具 Agent

### 第4周（6/22 – 6/30）—— 项目：文件系统助手
- [ ] 增加工具：`list_dir`、`read_file`、`write_file`
- [ ] 让 Agent 能够回答："列出当前目录所有 py 文件"
- [ ] 增加记忆：用列表保存最近 5 轮对话
- [ ] 里程碑项目：`file_agent.py`（200 行内）
- [ ] **【新增】工具调用自愈**：处理畸形 JSON、参数缺失、重复调用（借鉴 Reasonix）

## 🗓️ 第二阶段：RAG 与检索（7月 – 9月）
**目标**：让 Agent 能读取你的代码项目，回答问题。

### 第5周（7/1 – 7/7）—— 文本切块与向量化
- [ ] 学习 RAG 基本流程
- [ ] 安装 chromadb 或 lancedb
- [ ] 把一个 README.md 切块（chunk_size=500, overlap=50）
- [ ] 生成向量并存入数据库

### 第6周（7/8 – 7/14）—— 实现最简单的 RAG 问答
- [ ] 写一个检索函数：给定问题，返回最相关的 3 个块
- [ ] 将检索结果拼入 prompt 中，让模型基于这些内容回答
- [ ] 核心产出：一个"问我的 README"的 CLI 工具

### 第7周（7/15 – 7/21）—— 扫描本地代码库
- [ ] 用 `pathlib` 遍历项目目录，过滤 `.py` / `.kt` / `.java`
- [ ] 按文件切块，同时保留文件路径作为元数据
- [ ] 批量存入向量库（可花几分钟索引一个小项目）
- [ ] 测试："`login` 函数在哪个文件？"

### 第8周（7/22 – 7/31）—— 项目：代码问答 Agent
- [ ] 将上述功能包装为一个 CLI 命令：`ask "哪里定义了 User类"`
- [ ] 增加对话记忆，支持追问（如"它的构造函数参数呢？"）
- [ ] 里程碑项目：`code_qa_agent.py`，可回答中型代码库的结构问题
- [ ] **【新增】混合搜索**：grep（精确）+ RAG（语义）双路召回，按场景选（借鉴 Codex CLI）

### 第9周（8/1 – 8/14）—— RAG 参数调优
- [ ] 实验不同 chunk_size（256, 512, 1024）对回答质量的影响
- [ ] 实验不同 top_k（3, 5, 10）
- [ ] 尝试 rerank（用 `sentence-transformers` 或简单规则）
- [ ] 记录调优前后对比（至少 5 个问题）

### 第10周（8/15 – 8/31）—— 流式输出与成本观察
- [ ] 将 API 调用改为流式（stream=True）
- [ ] 打印每个请求的 token 用量，统计一次问答成本
- [ ] 尝试手动"缓存" system prompt 和项目规范（固定字符串）
- [ ] 核心产出：一份《我的 RAG Agent 成本分析表》
- [ ] **【新增】Append-Only 日志**：历史对话只追加不修改，模拟 Reasonix 的缓存命中率优化

## 🗓️ 第三阶段：Agent 框架与复杂任务（9月 – 11月）
**目标**：使用 LangGraph / LangChain 构建能完成多步任务的 Agent。

### 第11周（9/1 – 9/7）—— 引入 LangGraph
- [ ] 学习 LangGraph 的核心概念：StateGraph, Node, Edge
- [ ] 将之前的"天气 + 文件" Agent 用 LangGraph 重写
- [ ] 感受框架带来的状态管理优势

### 第12周（9/8 – 9/14）—— 增加"规划节点"
- [ ] 设计一个简单的 Plan-and-Execute Agent
- [ ] 第一个节点：让模型输出任务分解的步骤列表
- [ ] 第二个节点：依次执行每个步骤（调用工具）
- [ ] 核心产出：能自动"查找 log 文件 → 统计 ERROR 次数 → 生成报告"的 Agent

### 第13周（9/15 – 9/21）—— 多 Agent 协作初探
- [ ] 实现两个简单的 Agent：Coder（写代码）和 Reviewer（审查）
- [ ] 用一个"主管"Agent 决定何时调用谁
- [ ] 场景：生成一个排序算法，然后提出改进意见

### 第14周（9/22 – 9/30）—— 项目：开发助手 Agent
- [ ] Agent 能接收需求："为 utils.py 的 parse_config 函数添加异常处理"
- [ ] 流程：读取原代码 → 生成修改后代码 → 输出 diff
- [ ] 增加"安全检查"：修改前先备份原文件
- [ ] 里程碑项目：`dev_assistant_agent.py`
- [ ] **【新增】Plan 模式 + 审批门**：复杂任务执行前先出计划，用户确认后再动手（借鉴 Claude Code）

### 第15-16周（10月）—— 深入 LangGraph + 人工介入
- [ ] 学习 LangGraph 的 `interrupt` / `resume` 机制
- [ ] 实现一个"需要用户确认才修改文件"的 Agent
- [ ] 增加"回滚"功能（当修改出错时）
- [ ] 核心产出：一个半自动的代码修改助手
- [ ] **【新增】Checkpoint（自动快照）**：每次修改前自动保存文件快照，出问题可回滚（借鉴 Reasonix）

### 第17-18周（11月）—— 集成外部工具链
- [ ] 让 Agent 能够执行 Shell 命令（例如 `git status`, `pytest`）
- [ ] 实现"运行测试 → 如果失败 → 读取错误日志 → 尝试自动修复"
- [ ] 增加简单的权限控制（哪些命令允许执行）
- [ ] 里程碑项目：`auto_fix_agent.py`

## 🗓️ 第四阶段：成本优化与副业级项目（12月 – 2月过完年）
**目标**：让你的 Agent 低成本、稳定运行，并完成一个可展示的完整项目。

### 第19-20周（12月上旬）—— 缓存深入与 Prompt 优化
- [ ] 学习 Anthropic / DeepSeek 的"前缀缓存"API
- [ ] 重构你的 Agent，将 system prompt 和常用工具描述缓存
- [ ] 对比使用缓存前后的 token 消耗（记录至少 10 轮对话）
- [ ] 优化 project prompt（如 `AGENTS.md` 的内容结构）
- [ ] **【新增】成本仪表盘**：实时显示累计 token、缓存命中率、预估费用（借鉴 Reasonix 桌面端）

### 第21-22周（12月下旬）—— 项目：Telegram / Discord 代码助手
- [ ] 学习简单的 bot 框架（python-telegram-bot 或 discord.py）
- [ ] 将你之前开发的"代码问答 Agent"接入 bot
- [ ] 增加多用户隔离（每个用户的对话历史分开）
- [ ] 让 bot 能回答"这个代码片段有什么问题？"
- [ ] 核心产出：一个可公开访问的、能帮别人看代码的 Bot

### 第23-24周（1月上旬）—— 副业项目：个人知识库 RAG Agent
- [ ] 选择一个小众但你有积累的领域
- [ ] 将你的笔记、文档、已解决的问题集整理成知识库
- [ ] 搭建一个 Web UI（用 Streamlit 或 Gradio 快速实现）
- [ ] 让 Agent 不仅能检索，还能"帮你起草文档/周报"
- [ ] 核心产出：一个可以展示给面试官/客户的垂直 Agent

### 第25-26周（1月下旬）—— 性能与可靠性加固
- [ ] 为你的 Agent 添加错误重试机制（API 限流、超时）
- [ ] 增加单次对话的最大步数限制（防止无限循环）
- [ ] 实现简单的"人工接管"模式（当 Agent 不确定时求助用户）
- [ ] 写一份部署文档，让别人也能跑起来你的 Agent

### 第27-28周（2月上旬，过年前）—— 作品集整理与简历更新
- [ ] 挑选 3 个你最满意的项目（文件助手、代码问答、Telegram bot 或知识库）
- [ ] 为每个项目写 README：功能、架构图、如何运行、成本分析
- [ ] 录制一个 2 分钟的功能演示视频（或 GIF）
- [ ] 更新简历：突出"AI Agent 开发"相关项目经验，标注"熟练掌握 LangGraph、RAG、Function Calling"

### 第29周（过完年，2月中下旬）—— 求职/接单准备
- [ ] 在 LinkedIn / 主流招聘平台更新状态，开启"寻找机会"
- [ ] 准备 3 个常见的 AI Agent 面试问题答案（如"如何处理多步任务？"）
- [ ] 联系 2-3 个潜在的副业客户（如中小创业公司、AI 自媒体）
- [ ] 定下目标：春节后第一个月内，投出至少 20 份简历 或 发出 10 份副业 proposal
